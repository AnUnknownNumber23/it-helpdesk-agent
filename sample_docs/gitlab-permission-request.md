# GitLab 权限申请

## 权限级别说明

GitLab 项目权限应按最小权限原则授予。

- Guest：查看基本项目信息和 issue，不适合访问代码。
- Reporter：可查看代码、下载制品、查看流水线结果。
- Developer：可推送分支、创建合并请求、运行流水线。
- Maintainer：可管理分支保护、成员权限、CI/CD 设置。
- Owner：仅适用于顶层 group，负责最终权限审批和管理。

普通开发协作通常授予 Developer；只读审计或排障通常授予 Reporter。

## 申请前确认

申请人需要确认：

- GitLab 用户名或公司邮箱。
- 目标 group、project 或 repository 地址。
- 需要的权限级别。
- 访问目的和预计访问期限。
- 是否涉及生产代码、敏感配置或客户数据。

权限申请必须有项目 Owner 或直属经理审批。IT 不应绕过项目负责人直接授权。

## 跨团队访问

跨团队访问需要额外说明业务背景，例如联合开发、临时排障、安全审计或交接支持。

建议规则：

- 临时访问应设置到期时间。
- 只读需求优先授予 Reporter。
- 需要修改代码时授予 Developer，不默认授予 Maintainer。
- 涉及生产部署、密钥变量、受保护分支时必须由项目 Owner 明确审批。

## 常见问题

### 能看到项目但无法 push

可能原因：

- 当前权限只有 Reporter。
- 分支受保护，仅 Maintainer 或指定角色可 push。
- 本地 Git 使用了错误账号或旧 token。

### 无法 clone 仓库

可能原因：

- 未加入项目或上级 group。
- SSH key 未配置或已过期。
- HTTPS token 权限不足。

### CI/CD 变量不可见

这是常见的安全限制。敏感变量通常仅 Maintainer 可管理，且部分变量不会明文展示。

## 何时创建工单

请创建 IT 工单并附上审批记录：

- 需要新增、调整或移除 GitLab 项目权限。
- 用户无法访问已批准的项目。
- SSH key、access token 或 SSO 登录异常。
- 跨团队临时访问需要设置到期时间。
- 项目 Owner 离职或无法审批，需要权限继承处理。
