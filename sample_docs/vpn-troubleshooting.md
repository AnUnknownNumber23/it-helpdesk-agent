# VPN 连接故障排查

## 适用场景

员工无法连接公司 VPN、连接后无法访问内网系统、频繁断线，或提示账号、证书、网络相关错误时，可参考本文。

## 常见原因

- 本地网络不稳定，公共 Wi-Fi 限制 VPN 流量。
- VPN 客户端版本过旧或配置文件损坏。
- 域账号密码错误、账号被锁定或密码已过期。
- MFA 验证未通过，手机令牌不可用。
- DNS 缓存异常，导致内网域名无法解析。

## 基础排查步骤

1. 确认本地网络可正常访问公网网站。
2. 断开 VPN 后重新连接，避免重复快速点击连接按钮。
3. 检查系统时间是否准确，时间偏差可能导致证书或 MFA 校验失败。
4. 确认使用的是公司标准 VPN 客户端和最新配置文件。
5. 尝试重启 VPN 客户端；仍失败时重启电脑后再次连接。

## 刷新 DNS 缓存

Windows：

```powershell
ipconfig /flushdns
```

macOS：

```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

刷新后重新连接 VPN，并再次访问内网域名。

## 账号锁定处理

如果连续多次输入错误密码，域账号可能被锁定。典型现象包括：

- VPN 提示 authentication failed 或 bad credentials。
- 邮箱、GitLab、办公电脑登录也同时失败。
- 刚修改过密码，但仍使用旧密码连接 VPN。

处理建议：

1. 停止继续尝试登录，避免延长锁定时间。
2. 在自助密码平台确认密码状态。
3. 如果密码已过期或忘记密码，先完成密码重置。
4. 修改密码后，更新电脑、手机邮箱、VPN 客户端中保存的旧密码。

## 连接后无法访问内网

- 确认 VPN 显示已连接并分配到公司内网 IP。
- 使用内网系统域名访问，不要使用过期收藏夹地址。
- 尝试刷新 DNS 缓存。
- 如果只有单个系统不可访问，可能是该系统权限或服务异常。

## 何时创建工单

请创建 IT 工单并提供以下信息：

- VPN 客户端截图和完整错误提示。
- 发生时间、所在网络环境、是否更换网络测试过。
- 账号名、电脑系统版本、VPN 客户端版本。
- 是否刚修改密码、是否可通过 MFA。

以下情况应直接创建工单：

- 账号疑似锁定且自助平台无法解锁。
- VPN 证书过期或提示证书无效。
- 多名员工同时出现连接失败。
- 连接成功但所有内网系统均不可访问。
